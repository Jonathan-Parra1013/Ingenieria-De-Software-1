# ¿QUÉ ES?
- Es un software educativo innovador enfocado en el aprendizaje y refuerzo de las matemáticas para estudiantes de nivel básico y medio. La investigación aborda la problemática de la ansiedad matemática y la falta de herramientas personalizadas, factores que generan bloqueos cognitivos y desmotivación en el estudio autónomo.
- Junto con un aplicativo, que usa la integración de Modelos de Lenguaje Extensos (LLM) para ofrecer tutoría personalizada y el diseño del avatar "Dax", quien actúa como guía interactiva para humanizar la experiencia.

# PROBLEMATICA
- El problema no radica únicamente en la dificultad inherente de las matemáticas, sino en cómo el entorno digital presenta este conocimiento.Declaración del problema: Los estudiantes de primaria y secundaria, que a menudo ya experimentan ansiedad frente a las matemáticas, necesitan una plataforma digital de aprendizaje que sea limpia, predecible y que ofrezca asistencia inmediata.
- Asi mismo vemos el deficiente rendimiento academico de los estudiantes en la materia de Matematicas, la dificultad que presentan en la comprension de ciertos temas, y la poca posibilidad de acceso a herramientas de estudio autonomo para cada estudiante ya que dentro de la clase hay explicacion pero en ciertos casos, no es tan comprensible para cierta cantidad de estudiantes.

# PROPOSITO:
El propósito de este software es ofrecer a los usuarios una herramienta integral para el aprendizaje de las matemáticas, permitiéndoles adquirir conocimientos matemáticos de manera efectiva y atractiva. El software se diseñará para adaptarse a las necesidades individuales de los usuarios y fomentar su participación constante a través de diversas funcionalidades interactivas, motivando al usuario a dar seguimiento a su propio proceso de aprendizaje de forma autónoma. De esta manera, se promueve el interés y la constancia sin que la experiencia se perciba como una obligación, sino como una actividad dinámica y estimulante, adecuada tanto para personas que presentan dificultades en matemáticas como para los que ya son buenas en ellas y solo buscan seguir reforzando sus conocimientos en dicha materia mediante retroalimentación (cuestionarios, lecturas, ejercicios,etc) y  recursos interactivos.

# OBJETIVO GENERAL:
Diseñar y desarrollar un software educativo basado en el aprendizaje autónomo de las matemáticas en estudiantes de nivel básico y medio, con el fin de mitigar la ansiedad matemática y promover la constancia a través de tutorías personalizadas.

# OBJETIVOS ESPECIFICOS:
- Integración y aplicación de LLM'S para el aprendizaje personalizado para cada usuario.
- Software con interfaz web llamativa y totalmente funcional para el usuario, facil de entender y de interectatuar.
- Integración un sistema de seguimiento de progreso que permita al usuario visualizar sus avances y logros en tiempo real.
- Implementación de un software multiplataforma que permita el acceso funcional desde diferentes dispositivos, facilitando asi el aprendizaje en cualquier momento.

# SOLUCIÓN PROPUESTA:
- Lecciones Interactivas: Proporcionar lecciones interactivas que cubran aspectos de operaciones basicas, 
ejercicios practicos y comprensión matematica. 

- Ejercicios Prácticos: Ofrecer una variedad de ejercicios prácticos, 
con ejercicios respectivos para cada grado , y más, para fortalecer las habilidades matematicas.

- Evaluaciones y Seguimiento: Realizar evaluaciones regulares para medir el progreso del usuario y ajustar el contenido 
en función de sus necesidades. Proporcionar un seguimiento del progreso a lo largo del tiempo. 

- Interacción en Tiempo Real: Permitir a los usuarios practicar habilidades de conversación en tiempo real
con chatbots o socios de conversación virtual.

- Biblioteca de Recursos: Proporcionar acceso a una amplia biblioteca de recursos, que incluye vocabulario, ejemplos de diálogos, 
audios y videos auténticos en el tema objetivo.

- Personalización: Permitir a los usuarios personalizar su plan de aprendizaje según sus objetivos y preferencias, 
adaptando la dificultad y el contenido de acuerdo con su nivel.

# REQUISITOS FUNCIONALES
- Registro de usuarios: El sistema debe permitir que los estudiantes creen una cuenta para acceder a la plataforma.

- Inicio de sesión: Los usuarios deben poder ingresar con sus credenciales para acceder a su progreso.

- Diagnóstico inicial: El sistema debe realizar una evaluación diagnóstica para identificar el nivel del estudiante en matemáticas.

- Tutoría personalizada: El sistema debe proporcionar explicaciones y ayuda personalizada mediante IA (LLM).

- Interacción con el avatar Dax: El sistema debe mostrar un avatar virtual (Dax) que guíe, conforte y ayude al usuario durante el aprendizaje.

- Presentación de ejercicios matemáticos: El sistema debe mostrar ejercicios de matemáticas en el tema en el que vaya el estudiante o quiera, para practicar diferentes temas.

- Retroalimentación inmediata: El sistema debe informar si una respuesta es correcta o incorrecta, con su respectiva explicación de como se hace.

- Seguimiento del progreso: El sistema debe guardar y mostrar el progreso del estudiante (como una racha) y dar pequeños logros como motivación.

- Interfaz de navegación intuitiva: El sistema debe permitir que el usuario navegue fácilmente entre actividades y contenidos, de forma interactiva y que no sature al usuario.

# REQUISITOS NO FUNCIONALES
- Usabilidad: - La interfaz debe ser intuitiva, clara y fácil de usar.
              - Debe evitar sobrecarga visual para reducir la ansiedad matemática.
              - Debe ser lo más inclusiva posible.
  
- Rendimiento: El sistema debe responder rápidamente a las acciones del usuario.

- Disponibilidad: El software debe estar disponible para el acceso de los estudiantes en cualquier momento que lo necesiten.

- Seguridad: - Los datos de los usuarios deben almacenarse de forma segura.
             - El sistema debe proteger la información personal de los estudiantes.

- Escalabilidad: El sistema debe poder soportar más usuarios en el futuro.

- Compatibilidad: El software debe funcionar en diferentes dispositivos o navegadores

- Accesibilidad: El sistema debe ser accesible para estudiantes de primaria y secundaria con distintos niveles de conocimiento.

- Mantenibilidad: El sistema debe permitir actualizaciones y mejoras sin afectar su funcionamiento.
